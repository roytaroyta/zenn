電車で移動しているとき、カフェでコーヒーを飲んでいるとき。

「あ、あの画面のボタン、もう少し丸みを持たせてオレンジ色にしたいな」

と思い立つことはありませんか？

これまでは、家に帰って PC を開くか、重いラップトップを持ち歩くしかありませんでした。

しかし、もし **「Slack で呟くだけで、自宅の Mac が勝手にコードを書き換え、手元のスマホのアプリ画面がその場で更新される」** としたらどうでしょう？

本書では、自宅で常時稼働している Mac を「AI 専属プログラマー」に仕立て上げ、外出先から Slack で指示を飛ばすだけで React Native（Expo）のコードを自動改修させる、**完全自律型の ChatOps 環境**の構築手順を解説します。

## 本書で作るもの

- Slack の Bot（Bolt for JavaScript）が指示を受け取る
- Bot が `gemini-cli` を呼び出してコードを自律改修する
- Expo のホットリロードがスマホに即時反映される
- 外出先からでも Tailscale VPN 経由でリアルタイムに確認できる

完成すると、以下のような操作が可能になります。

```
外出先のスマホ
 ↓ Slack で「ログイン画面の背景を暗くして」と送信
自宅の Mac（常時起動中）
 ↓ Gemini CLI がコードを読んで書き換える
スマホの Expo Go
 ↓ すぐ変更が反映される
```

## 想定読者

- TypeScript と Node.js の基礎があれば十分です
- React Native や Expo の経験は問いません（仕組みを理解していれば OK）
- Slack アプリ開発は初めてでも問題ありません

## 本書の構成

- **システム構成と技術スタック** — 全体図と各コンポーネントの役割
- **Gemini CLI の採用理由と設定** — API キー不要の強力な運用方法
- **Tailscale × Expo によるどこでもライブプレビュー** — 外出先での確認を可能にする VPN 設定
- **Slack Bot 開発における 2 つの大きな罠と解決策** — 実運用でハマるポイントを事前に潰す
- **決定版 index.ts の全貌解説** — コードを一行ずつ丁寧に解説
- **運用・拡張のヒント** — 安定稼働させるためのテクニック

では、さっそく始めましょう。
