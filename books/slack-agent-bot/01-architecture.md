本環境を構築するためのコアとなる技術スタックと、それぞれの役割を整理します。

## 技術スタック一覧

| コンポーネント     | 技術                             | 役割                                                     |
| ------------------ | -------------------------------- | -------------------------------------------------------- |
| ホストマシン       | Mac（自宅で常時起動）            | 実際に Gemini を動かす「本体」                           |
| AIエンジン         | `@google/gemini-cli`             | コードの読み書きを自律的に行うエージェント               |
| Bot フレームワーク | Slack API ＋ Bolt for JavaScript | Slack からの指示を受け取る UI 層                         |
| モバイル開発       | React Native ＋ Expo             | コードを書き換えた結果をリアルタイムで確認できる開発環境 |
| VPN                | Tailscale                        | 外出先のスマホを自宅 Mac と同じネットワークに繋ぐ        |

## 処理のフロー

全体の処理の流れは以下のとおりです。

```
1. 外出先のスマホで Slack Bot に @メンション ＋ 指示を送る
2. 自宅 Mac で動く Node.js アプリ（Bolt）がメッセージを受信
3. Bot が gemini-cli を子プロセスとして起動し、プロジェクトディレクトリ内で指示を実行
4. Gemini がファイルを読み込み、自律的にコードを書き換えて保存
5. Expo のホットリロードが走り、Tailscale 経由で繋がったスマホの画面が即座に変わる
```

図で書くと以下のようになります。

```
[スマホ: Slack] ──指示──→ [Mac: Bolt Bot]
                                 │
                         gemini-cli を実行
                                 │
                         [プロジェクトのコード]
                                 │
                         ファイルを上書き保存
                                 │
                      [Expo ホットリロード]
                                 │
               [スマホ: Expo Go] ←── Tailscale VPN 経由
```

## なぜ「自宅 Mac に常時起動サーバー」か

クラウドのサーバーでも同様の仕組みは作れますが、本構成では自宅 Mac を使うことに意味があります。

- **Google AI Pro（Gemini Advanced）の認証をそのまま使える**
  API キー発行や従量課金管理が不要（詳細は次章）
- **プロジェクトのコードがそのまま置いてある**
  クラウドへのコード転送・同期の手間がない
- **Expo の開発サーバーも同じマシンで動かせる**
  端末の IP 管理がシンプルになる

## 事前に必要なもの

- Mac（常時起動できること）— デスクトップ機が理想
- Node.js 18 以上（推奨 20.x）
- Slack ワークスペース（管理者権限があるもの）
- Google アカウント（Google AI Pro に加入しているもの）
- Tailscale アカウント（無料で利用可能）
- スマホ（iOS / Android、Expo Go をインストール済み）

## 依存パッケージ

```json
{
  "dependencies": {
    "@slack/bolt": "^4.6.0",
    "dotenv": "^17.3.1"
  },
  "devDependencies": {
    "@types/node": "^25.0.0",
    "ts-node": "^10.9.2",
    "typescript": "^5.9.0"
  }
}
```

`@slack/bolt` が Slack との通信を担い、`dotenv` で環境変数を読み込みます。TypeScript で書くため `ts-node` を使って直接実行します。

## プロジェクト構成

本書のコードはシンプルに `index.ts` 1 ファイルで完結します。

```
slack-agent-bot/
├── index.ts         # Bot 本体（メイン処理はすべてここ）
├── package.json
├── tsconfig.json
└── .env             # Slack トークンや設定値（Git 管理外）
```
