---
title: "Overview — TypeScriptで自動投稿システム構築"
---

この本は、TypeScript で実装された自動投稿システムの設計から運用までを、実例（bluesky-post-bot）を参照しつつ丁寧に解説します。

目的

- システム全体の理解を得ること
- ローカル検証から CI 自動化、運用まで実践的に学ぶこと
- 生成モデル（Gemini）とソーシャル投稿 API を安全かつ再現性高く組み合わせる方法を示すこと

想定読者

- TypeScript の基礎知識を持ち、CI/CD と API の取り扱いに慣れたい開発者
- 自動投稿ワークフローを自分で構築・運用したい技術担当者

本書の構成（章タイトル）

- セットアップ — ローカル環境と依存関係の準備
- Secrets と環境変数 — キー管理と安全な運用
- ワークフロー設計 — GitHub Actions による自動化戦略
- ローカル実行 — ドライランとデバッグの手順
- ストレージと重複検出 — 投稿履歴管理と一意性の担保
- モニタリングとロギング — 可観測性とアラート設計
- プロンプト設計 — Gemini に良質な出力をさせる方法
- テストとドライラン — 自動テストと安全な本番移行
- トラブルシューティング — よくある問題と対処法
- 付録／変更履歴 — 重要な変更点の記録

運用の高レベルフロー

1. テーマやキーワードを決める（企画）
2. プロンプトテンプレートを使って Gemini に文章生成を依頼
3. 生成結果を整形・検証（禁止ワード・文字数・メディアの添付など）
4. 重複チェック（ハッシュ/投稿履歴照合）
5. 問題なければ Bluesky に投稿、成功なら投稿情報を保存

短い図（概念）:

```
[企画] -> [プロンプト] -> [Gemini] -> [整形・検証] -> [重複チェック] -> [投稿API] -> [保存/ログ]
```

この後の章では、上の各フェーズを順に詳細化します。まずはセットアップ章から順に読み進めてください。
