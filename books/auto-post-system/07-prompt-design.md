---
title: "07 プロンプト設計 — 要約 & 投稿文生成"
---

Gemini（あるいは別 LLM）へ投げるプロンプト設計の方針、テンプレート、注意点をまとめます。

項目:

- 要約テンプレート（短い見出し + 1 文要約など）
- 投稿用文面の長さ制限とトーン設定
- モデル温度や出力制約の物理的設定例
- 利用規約・著作権・フェアユースの観点での注意点

テスト:

- 同じ記事で複数のテンプレートを比較する方法（A/B テスト）

良質な生成を得るには、プロンプトを明確にし、出力を解析しやすい形で受け取ることが重要です。以下は実践的なガイドとテンプレート例です。

基本原則

- 目的を先に明示する（例: 「日本語で 280 文字以内の Bluesky 投稿を作る」）
- 出力フォーマットを厳密に指定する（JSON やセクション区切り）
- トーンや禁止事項を明示する（専門的/親しみやすい/絵文字可否 など）
- 文字数やハッシュタグ数などの制約を与える

プロンプトテンプレート（投稿生成）

```
あなたは日本語で自然で読みやすいソーシャル投稿文を生成するアシスタントです。
条件:
- 投稿は最大280文字に収めること。
- トーン: 親しみやすく、専門用語は必要最小限にする。
- 絵文字は最大2つまで許可。
- 結果は必ず JSON 形式で返す。キーは `text`（本文）、`hashtags`（配列）、`summary`（要約: 1 文）とする。

入力:
{{CONTENT}}

出力例:
{
	"text": "<投稿本文>",
	"hashtags": ["#tech", "#ai", "#tips"],
	"summary": "<1文要約>"
}
```

要約テンプレート（短縮）

```
あなたは日本語で記事の要点を 1 文で要約するアシスタントです。
条件: 15〜30 語で要約してください。
入力:
{{ARTICLE_TEXT}}
```

出力検証と整形

- 生成結果は JSON パースを行い、期待キーがなければ失敗として再生成する。
- 禁止ワードのチェック（ブラックリスト）を必ず通す。
- 文字数が超過している場合はトリムのルールを適用する（例: 末尾をカットして "…" を付与）。

プロンプトのチューニング

- Few-shot: 良い/悪いサンプルを 1〜3 個与えることで生成品質が向上することが多い
- Temperature の設定: `0.0-0.3`（決め打ち・再現性重視）、`0.5-0.9`（多様性重視）
- Max tokens: 投稿文なら低めに設定して制約違反を防ぐ

評価と改善

- 自動評価: 文字数、ハッシュタグ数、禁止ワードフィルタ、エンゲージメント予測スコア等
- 手動評価: キュレーターが目視チェックして不適切な出力がないか確認
- A/B テスト: 異なるテンプレートで生成した投稿を一定期間配信し、反応率を比較する

安全策（必須）

- 直接投稿する前に人が承認するフローを運用する（初期段階）
- 敏感なトピックは検出して生成を止める（ホットワード検出）
- モデルの利用規約やサービス提供者のポリシーを順守する
